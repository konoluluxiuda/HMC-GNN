# ETCM Graph Variants

This directory is generated by `preprocess_etcm_graph.py`.

The original `preprocess_paper_graph.py` is kept unchanged as Plan0 / old baseline. These graph variants implement Plan1 and use the ETCM-completed entity space.

## Protocol

- Disease-herb positives are read from `relation/herbTOdisease.csv`.
- For each disease, positives are split with seed `42` into 80% train and 20% test.
- Only train disease-herb positives are inserted into the graph.
- Herb-Herb and Disease-Disease Jaccard graphs are built from train disease-herb positives only.
- Base TCM/chemical relations are read from `herbTOchemical.csv`, `herbTOeffect.csv`, `herbTOflavor.csv`, and `herbTOchannelTropism.csv`.

## Gene-as-feature

Gene features are generated by `build_gene_features.py`.

The feature space uses `entities/gene_completed_etcm.csv` as gene columns:

- `node_disease_gene.pt`: sparse disease-gene multi-hot matrix.
- `node_herb_gene.pt`: sparse herb-gene multi-hot matrix.
- `node_gene_matrix.pt`: sparse merged matrix used by `train.py`.
- `gene_feature_map.csv`: gene column index to gene ID mapping.
- `gene_feature_summary.csv`: feature shape and nonzero counts.

Current shape for each variant:

```text
node_num × gene_num = 17683 × 5655
disease_gene_nnz = 68955
herb_gene_nnz = 46955
combined_gene_nnz = 115910
```

The model uses these features as gene-level semantic information rather than target-mechanism evidence:

```text
gene multi-hot -> Linear(5655, emb_dim) -> multimodal fusion
```

When `FUSION_MODE = 'gated'`, the enabled feature streams are fused by a node-level gate.
With the current all-feature setting, the gate uses:

```text
structure / attr / chem / disease_text / gene
```

When `FUSION_MODE = 'add'`, the projected gene feature is added to the fused node embedding.

Current `train.py` is prepared for the disease-disjoint split experiment:

```text
USE_DISEASE_SPLIT_GRAPH = True
ETCM_GRAPH_VARIANT = 'g4_gene_bridge_chemical'
USE_GENE_FEATURE = True
```

Old `recommendation_data` / `paper_graph_data` feature matrices are not reused because they use the old node order. The `g4_gene_bridge_chemical` optional feature matrices have been rebuilt against `g4_gene_bridge_chemical/node_map.csv`.

## Disease-Disjoint Split

`preprocess_disease_split_graph.py` builds a stricter recommendation protocol under:

```text
dataset/NEWHERB/disease_split_graph_data
```

The target triples are `herbTOdisease` labels. They are split by disease ID, not by random disease-herb edge:

```text
train diseases = 2162
val diseases   = 270
test diseases  = 271
train labels   = 203815
val labels     = 22547
test labels    = 22468
```

Only train diseases' herb labels are inserted into the graph. Validation/test disease-herb labels are kept only in `val_dict` and `test_dict` inside `rec_data.pt`.

Side-information relations are still allowed because they are assumed available at inference time:

```text
herbTOchemical.csv
herbTOeffect.csv
herbTOflavor.csv
herbTOchannelTropism.csv
diseaseTOgene_etcm_matched.csv
herbTOgene_etcm_matched.csv
herbTOchemical_etcm_matched.csv
```

The disease-disjoint `g4_gene_bridge_chemical/node_map.csv` is identical to `etcm_graph_data/g4_gene_bridge_chemical/node_map.csv`, so the rebuilt optional `.pt` files can be reused safely.

Disease split G4 summary:

## Notes

- With the current paper threshold, `Herb-Herb disease Jaccard` has 0 edges at `Jaccard >= 0.8, Top-10`. This is preserved intentionally for Plan1; tune the threshold in later Top-K sensitivity experiments.
- `candidate_herb_count` is 395 because only 395 herbs have disease-herb positive records in `herbTOdisease.csv`.
- `train.py` now has `USE_ETCM_GRAPH`, `USE_ETCM_LEAK_GRAPH`, and `USE_DISEASE_SPLIT_GRAPH` switches.
- The current prepared run uses `disease_split_graph_data/g4_gene_bridge_chemical` with rebuilt g4 feature matrices.
- Existing non-gene feature matrices under `recommendation_data` / `paper_graph_data` still use the old node order. Use the rebuilt matrices under `g4_gene_bridge_chemical` for the current ETCM experiment.
