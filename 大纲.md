HMC-GNN 修订稿大纲
0. 核心定位
HMC-GNN: Gene-Overlap Enhanced Multi-View Heterogeneous Graph Learning for Disease-Driven Herb Recommendation  
本稿件应将 HMC-GNN 呈现为一个疾病驱动的中药推荐框架，该框架构建于一个多模态异构疾病-中药知识图谱之上。核心问题不仅仅是疾病输入与症状集输入不同，而是疾病-中药监督信号稀疏，难以仅凭直接标签进行泛化。因此，修订后的论文应聚焦于如何将外部生物学和多模态知识转化为有用的推荐结构。
最终稿件应仅保留三个主要创新点：
1. 基因重叠引导的局部语义图增强。
2. 互补的局部-全局-语义视图学习。
3. 带有对比对齐的门控多模态语义融合。
疾病标签不交叠的评估、阈值敏感度分析及泄漏检查是重要的实验严谨性体现，但不应被写成主要的方法论创新。
1. 主要工作与创新点
1.1 基因重叠引导的局部语义图增强
这是修订稿的首要创新点。
核心思想是基因关联不应仅用作节点侧特征或额外的基因节点。相反，疾病-基因和中药-基因关联可以诱导出同类型的语义关系：
● 共享关联基因的疾病可能具有相关的生物学机制；
● 共享关联基因的中药可能具有相关的治疗或分子效应。
因此，HMC-GNN 计算同类型节点之间的共享基因相似度，并利用它来构建疾病-疾病和中药-中药的局部语义边。相似度计算方式如下：
J(i,j) = |G_i ∩ G_j| / |G_i ∪ G_j|
其中 G_i 和 G_j 分别是两个疾病或两个中药的关联基因集。
图增强流程为：
疾病-基因 / 中药-基因关联
-> 共享基因筛选
-> Jaccard 相似度计算
-> 有界邻域稀疏化
-> 疾病-疾病和中药-中药局部语义边
当前主要配置：
图变体：g4_gene_bridge_chemical_gene_jaccard
稀疏化：基于百分位
分数百分位：90.0
语义残差：启用，权重 = 0.3
关系 dropout：禁用
边加权 gene-jaccard RGCN：禁用
构图配置
graph variant: g4_gene_bridge_chemical_gene_jaccard
gene-jaccard sparsification: percentile
score percentile: 90.0
gene-jaccard edge weighting: disabled
chemicalTOgene: not included in the main graph
模型配置
semantic residual: enabled, beta = 0.3
relation dropout: disabled
branch fusion: gate
local/global/semantic branches: enabled
为什么这是创新的：
● 它将基因关联从静态辅助信息转化为图拓扑。
● 它创建了局部同类型语义邻域，弥补了疾病-中药标签稀疏的问题。
● 它避免了仅仅依赖疾病-中药直接共现。
● 它提供了一种有生物学依据但仍面向推荐的图增强机制。
不应过度声称的：
● 不要声称因果性生物医学机制。
● 不要声称所选百分位是普遍最优的。
● 不要声称仅凭基因重叠就能解释临床疗效。
建议的稿件措辞：
我们没有仅仅将基因关联视为辅助节点属性，而是利用基因重叠来诱导疾病和中药之间的局部语义邻域。这将外部生物学标注转化为与推荐相关的同类型图结构，使模型能够将治疗信号传播到稀疏的疾病-中药标签之外。
1.2 互补的局部-全局-语义视图学习
这是第二个主要创新点。
之前的统一图框架对于当前的实现来说过于简单。最终的 HMC-GNN 应被描述为一个互补视图模型，包含三个分离的视图：
1. 局部视图：
直接的推荐标签和局部相似性关系。
局部视图由两类边组成：
1. 推荐监督边：
  treats_disease / rev_treats_disease；
2. 同类型局部相似边：
   disease_herb_jaccard、herb_disease_jaccard、disease_gene_jaccard、herb_gene_jaccard。
2. 全局视图：
全局视图包括中药-成分、中药-功效、中药-属性/药味、中药-归经、疾病-基因、中药-基因等外部异构知识关系。当前主模型不直接使用 chemicalTOgene 边，以避免将成分-基因关系解释为靶向机制。
3. 语义视图：
纯节点级多模态语义表示，不进行图传播。语义视图由属性/成分 multi-hot、化学 dense/fingerprint、疾病文本和 gene feature 经投影后形成。
为什么这是创新的：
● 它避免了将治疗相互作用、外部知识和原始语义特征强制放入一个统一的传播通道。
● 它允许局部疾病/中药相似性和全局异构知识贡献互补信息。
● 它减少了高度异构图中的语义干扰。
● 与单一的整体图相比，它使模型更易于解释。
当前实现：
全局分支：在外部知识边上的 RGCN
局部分支：在疾病-中药及同类型局部语义边上的 RGCN
语义分支：投影的多模态节点语义，无 RGCN
分支融合：门控
语义残差：语义分支也以 0.3 的权重注入到图分支中
语义残差应被描述为互补视图学习的扩展：
z_global = RGCN_global(x) + beta * z_semantic
z_local  = RGCN_local(x)  + beta * z_semantic
其中在当前主要配置中 beta = 0.3。
建议的稿件措辞：
我们将异构图解耦为局部、全局和语义视图。局部视图捕获推荐特定和基因重叠诱导的同类型关系，全局视图捕获外部异构知识，语义视图保留原始多模态节点信息。一条语义残差路径进一步将节点级语义注回图传播表示中，以减轻多跳聚合过程中的语义稀释。
1.3 门控多模态语义融合与对比对齐
这是第三个支撑性创新。
修订稿应强调中药节点和疾病节点不共享相同的模态结构，因此需要对结构、属性、化学、文本和基因等不同语义流进行可学习融合。当前实验表明，门控融合明显优于直接相加；type-aware gate 是更符合中药/疾病模态非对称性的最终实现，但不应过度声称其相对 shared gate 带来巨大提升。
中药侧流：
结构嵌入
中药属性特征
化学特征
基因特征
疾病侧流：
结构嵌入
疾病文本特征
基因特征
类型感知门控融合：
Herb: structure + TCM attribute/component multi-hot + chemical dense/fingerprint + gene feature
Disease: structure + disease text + gene feature
 视图级门控融合  
z = Gate(z_local, z_global, z_semantic)
View-level gate:
z_h = Gate_h^{view}(z_h^{local}, z_h^{global}, z_h^{sem})
z_d = Gate_d^{view}(z_d^{local}, z_d^{global}, z_d^{sem})
优化结合了推荐和辅助对齐目标：
L = L_BPR
  + lambda_g L_graph_SSL
  + lambda_c L_cross_modal
  + lambda_p L_property_chemical
为何重要：
● 中药和疾病节点具有不对称的特征可用性。
● 化学信号对中药表示尤其重要。
● 文本和基因特征有助于疾病表示。
● 对比和对齐损失防止图嵌入偏离有意义的模态特定语义。
建议的稿件措辞：
由于中药和疾病关联不同的语义模态，HMC-GNN 采用门控多模态融合。中药节点整合结构、传统属性、化学和基因信号，而疾病节点整合结构、文本和基因信号。最终实现使用 type-aware gate 以匹配中药/疾病的模态非对称性；融合后的表示进一步通过图自监督和跨模态语义对齐目标进行正则化。
2. 修订稿结构
2.1 摘要
摘要应围绕以下逻辑重写：
1. 疾病驱动的中药推荐面临疾病-中药标签稀疏的问题。
2. 现有方法常将外部知识用作辅助特征或统一图边，但未明确将基因关联转化为局部语义拓扑。
3. HMC-GNN 引入了基因重叠引导的局部语义图增强。
4. HMC-GNN 学习互补的局部、全局和语义视图，并采用门控多模态融合。
5. 在疾病标签不交叠协议下的实验展示了推荐性能的提升。
建议的摘要框架：
疾病驱动中药推荐旨在为给定疾病排序候选中药，但直接的疾病-中药监督稀疏且难以泛化。为解决此问题，我们提出 HMC-GNN，一个基因感知的异构推荐框架。HMC-GNN 并未将基因关联仅用作辅助特征，而是基于共享基因重叠构建疾病-疾病和中药-中药的局部语义边。该模型进一步将图分解为互补的局部、全局和语义视图，并通过门控融合整合非对称的中药和疾病模态。语义残差路径在图传播过程中保留原始多模态语义，而多目标自监督对齐改善了表示的一致性。在我们构建的疾病-中药数据集上，于疾病标签不交叠协议下进行的实验表明，HMC-GNN 在 top-K 中药推荐任务上取得稳定提升。
2.2 引言
2.2.1 背景
引入疾病驱动的中药推荐，将其作为计算传统医学中一项重要的推荐任务。输入是疾病，输出是候选中药的排序列表。
关键挑战：
1. 疾病-中药标签稀疏，仅依赖直接 disease-herb 边泛化不足；
2. 外部 gene 信息常被作为节点属性或普通异构边，未被转化为推荐相关的局部语义拓扑；
3. 统一图传播容易混合局部推荐关系、全局异构知识和原始语义特征，造成语义干扰。
不要主要将新颖性框定为“疾病输入而非症状输入”。更好的框架是：
核心挑战是如何将稀疏的疾病标签和外部生物医学知识转化为有效的推荐结构。
2.2.2 现有工作的局限性
将先前方法归纳为三个局限性：
1. 共现与基于矩阵的方法：
它们捕获直接的统计关联，但难以处理高阶异构知识。
2. 统一 KG/GNN 方法：
它们聚合异构信息，但局部推荐交互、外部知识关系和原始语义特征在单一图中传播时可能相互干扰。
3. 多模态特征方法：
它们使用化学、文本或生物学特征，但常将基因关联作为辅助信息，而非将其转化为局部图结构。
2.2.3 提出的思路
陈述核心见解：
基因重叠可以为疾病和中药提供局部语义邻域。这些局部邻域应与全局异构知识和原始节点语义分开学习。
2.2.4 贡献
精确列出三个主要贡献：
1. 我们提出了一种基因重叠引导的局部语义图增强策略，根据共享的基因关联构建疾病-疾病和中药-中药的同类型边。
2. 我们设计了一个互补的局部-全局-语义视图学习框架，该框架分离了推荐特定的局部拓扑、外部异构知识以及原始多模态节点语义，并通过语义残差路径在图传播过程中保留节点级信息。
3. 我们引入了门控多模态语义融合与多目标对比对齐，使中药和疾病能够整合非对称的模态流，同时保持图、化学和属性级的语义一致性。
然后单独提及实验严谨性：
我们在疾病标签不交叠协议下评估模型，并进行消融、敏感度和诊断分析，以验证各组件的贡献。
不要将疾病标签不交叠协议列为创新点。

z_local  = z_local  + beta * z_sem

2.7.3 基因重叠分析
聚焦于：
● 基因重叠改善了同类型局部语义结构；
● 百分位稀疏化优于过于宽松或过于严格的设置；
● 连续的边权重未能优于稀疏的二值局部语义边。
不要声称：
● 基因重叠证明了生物学因果关系；
● 所选阈值是普遍适用的。
2.8 消融研究
说明：不再把 graph variant ablation 作为主消融，因为 g0/g3/g4 与 gene source、实体关系和 view ablation 存在明显重叠。主文消融应围绕“实体/关系贡献、基因重叠来源、互补视图、融合方式和优化目标”展开。

1. Entity/Relation ablation
   目的：说明不同实体类型和外部知识关系对推荐性能的贡献，避免审稿人认为性能只来自某一类边或某一类特征。
   建议设置：
   Full model
   w/o gene relations/features
   w/o chemical nodes/relations/features
   w/o effect relations
   w/o property + meridian relations/features
   w/o same-type gene-overlap edges
   说明：如果实现时难以完全删除实体节点，可采用“删除对应边 + 删除对应节点特征”的方式，并在实验设置中说明。

2. Gene-overlap source ablation
   目的：证明提升来自疾病侧和中药侧同类型语义邻域的互补，而不是某一侧单独贡献。
   herb_gene_jaccard only
   disease_gene_jaccard only
   herb + disease gene_jaccard

3. Multi-view ablation
   目的：验证 local/global/semantic 三个视图是否互补。
   w/o local
   w/o global
   w/o semantic
   local + global
   global + semantic
   local + semantic
   local + global + semantic
   注意：如果某些组合与已有设置等价，要在表注中说明，避免重复命名。

4. Fusion ablation
   目的：证明可学习门控融合优于简单相加，同时谨慎说明 type-aware gate 的作用。
   feature add
   shared gate
   type-aware gate
   branch sum/add
   branch gate
   结论表述：门控融合是关键；type-aware gate 是语义上更合理的最终实现，但不要声称它显著优于 shared gate。

5. Semantic residual ablation
   beta = 0 / 0.1 / 0.2 / 0.3 / 0.4
   目的：说明语义残差能够缓解图传播中的语义稀释，并确认 beta=0.3 是验证集选择的主配置。

6. SSL and alignment ablation
   w/o GraphSSL
   w/o CrossModal
   w/o PropertyChemical
   w/o all auxiliary losses
   说明：w/o CrossModal 已确认为干净消融，即仅去除跨模态 SSL 权重，而不是同时删除化学特征流。

7. Gene-overlap diagnostics
   edge-weighted vs binary
   percentile sensitivity
   adaptive sparsification if reported
   说明：这些作为鲁棒性和诊断分析，不作为主要创新点。
2.9 鲁棒性与诊断分析
2.9.1 百分位敏感度
当前结果：
85: F1@10=0.2277, NDCG@10=0.7029
90: F1@10=0.2657, NDCG@10=0.7829
95: F1@10=0.2479, NDCG@10=0.7437
解读：
85 过于宽松，95 过于严格，而在当前的运行中，90 提供了最佳的验证/测试平衡。
2.9.2 边加权 Gene-Jaccard 诊断
当前结论：
Jaccard 边加权未优于稀疏的二值基因重叠边。
这支持了如下主张：主要价值在于构建有用的语义邻域，而非简单地按相似度分数缩放消息。

2.10 讨论
讨论四点：
1. 为什么基因重叠局部语义边能改善稀疏的疾病-中药推荐。
2. 为什么局部/全局/语义解耦优于单一统一图。
3. 为什么语义残差有助于在图传播中保留原始节点语义。
4. 局限性与未来工作。
局限性：
● 基因重叠并不意味着因果性治疗机制；
● 阈值/百分位是经过验证集选择的；
● 二值疾病-中药标签不编码中药重要性；
● 外部基线必须在相同协议下重新运行以进行严格比较；
● 目前仅使用了一个数据集。
● 当前基因重叠增强仅建模基于共享基因的两两相似关系，尚未刻画基因通路层级或基因-基因调控关系。
2.11 结论
结论应简洁：
我们提出了 HMC-GNN，一个疾病驱动的中药推荐模型，它将基因关联转化为局部语义图结构，学习互补的局部/全局/语义视图，并通过类型感知融合和对比对齐整合非对称的中药/疾病模态。在疾病标签不交叠协议下的实验表明，基因重叠引导的局部语义增强和语义残差视图学习改善了 top-K 中药推荐。未来工作将把该框架扩展到多个数据集、更丰富的临床标签以及更具自适应性的图稀疏化方法。
3. 用于稿件的最终贡献声明
在最终论文中使用此版本：
本工作的主要贡献总结如下。

首先，我们为疾病驱动中药推荐提出了一种基因重叠引导的局部语义图增强策略。通过计算同类型节点间的共享基因 Jaccard 相似度，我们构建了疾病-疾病和中药-中药语义边，将生物学基因关联转化为与推荐相关的局部拓扑。

其次，我们设计了一个互补的局部-全局-语义视图学习框架。局部视图捕获疾病-中药交互和基因重叠语义邻域，全局视图捕获异构外部知识，语义视图保留原始多模态节点信息。一条语义残差路径进一步减轻了图传播过程中的语义稀释。

第三，我们为非对称的中药和疾病表示引入了门控多模态融合和对比对齐。中药整合了结构、传统属性、化学和基因特征，而疾病整合了结构、文本和基因特征。模型通过排序、图自监督、跨模态化学对齐以及属性-化学对齐目标进行优化。
